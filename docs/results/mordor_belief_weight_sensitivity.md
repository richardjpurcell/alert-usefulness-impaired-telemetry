# Mordor belief weight sensitivity

This note records a small sensitivity check on the first Mordor
belief/usefulness bridge.

## Purpose

The Mordor healthy run produced mostly redundant updates because the current
scalar host-compromise belief quickly saturated at the configured maximum
probability of 0.999.

This experiment asks whether reducing belief-update weights substantially
changes that behaviour.

## Script

Generated by:

```bash
python scripts/run_mordor_belief_weight_sensitivity.py
```

Output:

- `outputs/tables/mordor_belief_weight_sensitivity.csv`

The CSV output is reproducible and is not tracked.

## Configurations

Four healthy-mode belief configurations were compared:

| Configuration | Auth weight | Endpoint weight | Network weight | Score weight |
|---|---:|---:|---:|---:|
| default | 0.080 | 0.180 | 0.120 | 0.2500 |
| half_weights | 0.040 | 0.090 | 0.060 | 0.1250 |
| quarter_weights | 0.020 | 0.045 | 0.030 | 0.0625 |
| low_weights | 0.010 | 0.025 | 0.015 | 0.0350 |

## Results

| Configuration | Useful events | Redundant events | Misleading events | Uninformative events | Saturated update fraction |
|---|---:|---:|---:|---:|---:|
| default | 7 | 5,986 | 33 | 0 | 0.993 |
| half_weights | 7 | 5,957 | 62 | 0 | 0.989 |
| quarter_weights | 7 | 5,910 | 109 | 0 | 0.981 |
| low_weights | 30 | 5,843 | 110 | 43 | 0.968 |

## Interpretation

Reducing the belief-update weights changes the label distribution, but only
modestly. The lowest-weight configuration increases useful events from 7 to 30
and introduces 43 uninformative events, but 5,843 of 6,026 healthy delivered
events are still labelled redundant.

The saturated update fraction decreases from 0.993 under the default
configuration to 0.968 under the lowest-weight configuration. This confirms that
the redundancy pattern is partly weight-sensitive, but it is not explained by
weights alone.

The larger driver is the repeated-event structure of the Mordor-derived
telemetry under the current scalar host-belief abstraction. Once a host belief
saturates, repeated events for that host cannot materially change belief or
entropy, even when the event stream remains meaningful in a security-analysis
sense.

## Main point

Lower belief weights delay saturation, but they do not remove the saturation
problem. The next modelling step should probably examine event aggregation,
diminishing returns, confidence tracking, or episode-level usefulness rather
than simply tuning weights.

## Interpretation boundary

This is a sensitivity check, not a proposed replacement for the default model.
The default model remains unchanged.
