# Mordor belief interpretation check

This note explains why the first Mordor healthy belief/usefulness run produced
mostly redundant updates.

## Context

The first Mordor belief/usefulness run reported the following healthy-mode
diagnostics:

- useful events: 7
- redundant events: 5,986
- misleading events: 33

This check regenerates the healthy-mode Mordor diagnostics and summarizes the
event-level belief/update behaviour by usefulness label.

## Diagnostic outputs

Generated by:

```bash
python scripts/inspect_mordor_belief_usefulness.py
```

Output tables:

- `outputs/tables/mordor_healthy_usefulness_interpretation.csv`
- `outputs/tables/mordor_healthy_usefulness_diagnostic_samples.csv`

The CSV outputs are reproducible generated outputs and are not tracked.

## Label-level interpretation

| Label | Events | Mean event score | Mean belief delta | Mean entropy delta | Mean belief before | Mean belief after |
|---|---:|---:|---:|---:|---:|---:|
| misleading | 33 | 0.355 | 0.0863 | -0.0250 | 0.5212 | 0.6075 |
| redundant | 5,986 | 0.591 | 0.0000 | 0.0000 | 0.9990 | 0.9990 |
| useful | 7 | 0.371 | 0.0095 | -0.0726 | 0.9895 | 0.9990 |

## Explanation

Most healthy Mordor events are labelled redundant because the current ASTRA
belief abstraction quickly saturates host compromise belief at the configured
maximum probability of 0.999. Once belief is saturated, later events for the
same host cannot materially change belief or entropy. They are therefore
classified as redundant by the current usefulness diagnostic.

The 7 useful events are not a large class of repeatedly useful alerts. They are
primarily the last small updates that move already-high beliefs into saturation.

The 33 misleading events occur when delivered events increase the defender's
compromise belief while the delivery-time state abstraction is not compromised.
Under the current diagnostic, that increases belief error and is labelled
misleading.

## Interpretation boundary

This result should not be treated as evidence that only 7 Mordor events are
operationally useful. It is evidence about the current ASTRA abstraction:

- repeated Mordor-derived events can rapidly saturate scalar host belief;
- once saturated, additional events are naturally labelled redundant;
- useful/redundant/misleading labels are sensitive to the current state mapping,
  probability clipping, and per-event update rule.

## Main point

The current behaviour is explainable and internally consistent. The next
modelling question is whether ASTRA should keep this saturation behaviour as a
deliberately simple first abstraction, or introduce a less saturating belief
model for repeated real telemetry.
